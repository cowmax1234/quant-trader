# Quant Trader — 프로젝트 요약 (수석 퀀트 트레이더 AI 피드백용)

> Upbit·Binance API와 AI(Gemini)를 활용한 **비트코인 자동매매/대시보드** 개인 프로젝트의 현재 상태를 구조적으로 정리한 문서입니다.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **목적** | Upbit·Binance 잔고 조회 + 시장 데이터 기반 AI 채팅(시장 인사이트) 제공. 향후 자동매매 확장 예정. |
| **성격** | 개인용. API 키·시크릿은 코드에 넣지 않고 `.env`만 사용. |
| **실행** | 로컬: `npm run dev` (localhost:3003) / 배포: AWS EC2 (15.135.65.106:3003), PM2로 상시 실행. |

## 3. 기술 스택

| 구분 | 기술 |
|------|------|
| **Frontend** | Next.js 14 (App Router), React 18, TypeScript. CSS는 인라인/globals.css (별도 UI 라이브러리 없음). |
| **Backend** | Next.js API Routes (Route Handlers). Node.js 서버에서만 Upbit/Binance/Gemini 호출. |
| **DB** | 없음. |
| **외부 API** | Upbit REST (JWT), Binance Spot REST (HMAC), CoinGecko 공개 API, Google Gemini API (`@google/genai`, 모델: `gemini-1.5-flash`). |
| **배포** | AWS EC2 (Ubuntu), PM2. 로컬에서 프로덕션 빌드 후 `.next`를 zip으로 올려 서버에서 압축 해제·권한 수정 후 `pm2 restart` (서버 t2.micro에서 `npm run build` 시 메모리 부족으로 실패). |

## 4. 구현된 핵심 기능

### 4.1 자산 조회 (Upbit · Binance)

- **Upbit**  
  - `lib/upbit.ts`: JWT 생성 후 `GET /v1/accounts` 호출.  
  - `GET /api/upbit/accounts`: 보유 코인별 balance, locked, avg_buy_price 등 반환.  
  - API 키 미설정 시 `configured: false`로 응답.

- **Binance**  
  - `lib/binance.ts`: HMAC 서명 후 `GET /api/v3/account` 호출.  
  - `GET /api/binance/account`: Spot 잔고 목록 반환.  
  - API 키 미설정 시 `configured: false`로 응답.

- **자산 현황 페이지** (`/assets`): 위 두 API를 호출해 테이블로 표시. “새로고침”으로 재요청.

### 4.2 시장 데이터 (공개 API)

- **marketContext.ts**  
  - Upbit 공개: `KRW-BTC`, `KRW-USDT` 시세.  
  - CoinGecko 공개: BTC/USD, 24h 변동률.  
  - `getMarketSnapshot()`: 위 데이터를 묶어 스냅샷 반환.  
  - `getMarketContextText()`: Gemini 시스템 프롬프트에 넣을 텍스트 생성.

- **GET /api/context**: 현재 시장 스냅샷 JSON (btcKrw, btcUsd, btcChange24h, usdtKrw, fetchedAt).  
  - 채팅 페이지 상단 “현재 시장 요약” 및 채팅 API의 컨텍스트로 사용.

### 4.3 시장 인사이트 채팅 (Gemini)

- **POST /api/chat**  
  - 요청: `{ message: string, history: ChatMessage[] }`.  
  - 서버에서 `getMarketContextText()`로 시장 컨텍스트를 가져와 시스템 프롬프트에 포함.  
  - `@google/genai`로 `gemini-1.5-flash` 호출, 스트리밍 없이 한 번에 응답.  
  - 응답: `{ text: string }`.  
  - CORS: 로컬(localhost:3003)에서 EC2 API 호출 시 OPTIONS + CORS 헤더 처리.

- **채팅 페이지** (`/chat`): 시장 요약 표시 + 사용자 입력 → `/api/chat` 호출 → AI 응답 표시.  
  - `NEXT_PUBLIC_DEV_API_URL`이 설정되면 브라우저가 EC2의 `/api/chat`, `/api/context`를 호출.

### 4.4 로컬 vs 배포 동작

- **getApiBase()**  
  - `NEXT_PUBLIC_DEV_API_URL`이 있으면 → 브라우저가 해당 URL(예: EC2)로 `/api/*` 요청.  
  - 없으면 → 상대 경로로 요청해 같은 오리진(로컬 서버)이 처리.

- **실제 잔고·채팅**  
  - Upbit는 API IP 제한으로 **EC2 IP만 허용**되어, 실제 잔고는 배포 URL(15.135.65.106:3003)에서만 정상 동작.  
  - 로컬은 UI/기능 개발용. 채팅은 로컬 `.env`에 `GEMINI_API_KEY`만 넣고 `NEXT_PUBLIC_DEV_API_URL`을 비우면 로컬 API로 동작.

## 5. 미완성·제약·알려진 이슈

| 구분 | 내용 |
|------|------|
| **자동매매** | 미구현. 현재는 잔고 조회 + 시장 인사이트 채팅만 제공. |
| **Upbit 로컬 테스트** | Upbit API IP 화이트리스트가 EC2만 허용되어, 로컬에서는 잔고 API가 차단될 수 있음. |
| **EC2 빌드** | t2.micro 메모리 부족으로 서버에서 `npm run build` 실패(SIGBUS). 배포는 로컬 빌드 → `.next` zip → 서버 업로드 → unzip → `sudo chown`/`chmod` → `pm2 restart` 방식 사용. |
| **Gemini 할당량** | 무료 한도(일/분당 요청·토큰) 초과 시 429. 현재는 `gemini-1.5-flash` 사용(기존 2.0-flash 무료 한도 0 이슈 회피). |
| **서버 폴더 구조** | EC2 상 경로가 `~/quant_trader/quant_trader`로 중첩되어 있음. 정리 가능. |
| **에러/미완성** | DB 없음, 로그/모니터링 체계 없음, 자동매매·백테스트·리스크 관리 미구현. |

## 6. 환경 변수 (.env)

| 변수 | 용도 |
|------|------|
| `UPBIT_ACCESS_KEY`, `UPBIT_SECRET_KEY` | Upbit API 인증. |
| `BINANCE_API_KEY`, `BINANCE_API_SECRET` | Binance API 인증. |
| `GEMINI_API_KEY` | Gemini API (시장 인사이트 채팅). |
| `NEXT_PUBLIC_DEV_API_URL` | (선택) 로컬 개발 시 브라우저가 API를 보낼 주소. 예: `http://15.135.65.106:3003`. |
