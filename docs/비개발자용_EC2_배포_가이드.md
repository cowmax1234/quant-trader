# 비개발자용 — EC2에 대시보드 올리기 (복사·붙여넣기만 하면 됨)

개발을 몰라도 아래 순서대로 **그대로 따라 하시면** 됩니다.  
업비트에 **15.135.65.106** 등록까지 하셨다면, 여기서부터 진행하세요.

---

## 준비물 (본인 PC에 있어야 하는 것)

- **my-quant-key.pem** (EC2 만들 때 받은 키 파일)
- **quant_trader** 폴더 전체 (지금 Cursor로 열어 둔 이 프로젝트)
- **업비트 API 키**  
  - Access Key  
  - Secret Key  
  (업비트 마이페이지 → Open API 관리에서 확인 가능)

---

## 1단계: WinSCP로 프로젝트를 서버에 복사하기

서버에 코드를 넣는 가장 쉬운 방법입니다.

1. **WinSCP 설치**  
   - https://winscp.net/eng/download.php 에서 **Download WinSCP**  
   - 설치 후 실행

2. **접속 정보 입력**  
   - **파일 프로토콜**: SFTP  
   - **호스트 이름**: `15.135.65.106`  
   - **포트**: 22  
   - **사용자 이름**: `ubuntu`  
   - **암호**: 비워 두기  
   - **고급** 클릭 → 왼쪽에서 **인증** 선택 → **개인 키 파일**에 `my-quant-key.pem` 파일 선택  
   - **"OpenSSH 개인 키를 PuTTY 형식으로 변환하시겠습니까?"** 창이 뜨면 → **예** 클릭  
   - 저장 위치 정해 주고(예: .pem과 같은 폴더) **저장** → `my-quant-key.ppk` 같은 이름으로 저장됨  
   - 다시 **개인 키 파일** 칸에서 방금 저장한 **.ppk** 파일을 선택 → **확인**  
   - **로그인** 클릭  
   - "호스트 키를 신뢰하시겠습니까?" 나오면 **예**

3. **프로젝트 폴더 복사**  
   - 왼쪽: **본인 PC** — `quant_trader` 폴더가 있는 위치로 이동 (예: `문서`)  
   - 오른쪽: **서버** — 빈 화면(홈 폴더)  
   - 왼쪽에서 **quant_trader** 폴더를 **드래그해서** 오른쪽으로 놓기  
   - 복사가 끝날 때까지 기다리기

4. WinSCP는 **끄지 말고** 그대로 두거나, 복사만 끝나면 꺼도 됩니다.

## 2단계: 서버에 들어가기 (SSH)

1. **PowerShell** 열기  
   - Windows 검색에서 "PowerShell" 입력 후 실행

2. **키 파일이 있는 폴더로 이동**  
   - 예: 키가 `다운로드`에 있으면 아래 중 하나만 **한 줄** 입력 후 Enter  
   ```powershell
   cd $env:USERPROFILE\Downloads
   ```  
   - 키가 `바탕화면`이면:  
   ```powershell
   cd $env:USERPROFILE\Desktop
   ```

3. **서버 접속** (아래 한 줄 통째로 복사해서 붙여넣기 후 Enter)  
   ```powershell
   ssh -i "my-quant-key.pem" ubuntu@15.135.65.106
   ```  
   - "Are you sure you want to continue connecting?" 나오면 **yes** 입력 후 Enter  
   - 맨 앞에 `ubuntu@ip-172-31-26-172:~$` 처럼 나오면 **접속 성공**입니다.

---

## 3단계: 서버 안에서 .env 파일 만들기

API 키를 서버에만 넣는 단계입니다. **한 줄씩** 입력하고 Enter 하세요.

1. **프로젝트 폴더로 이동**  
   ```bash
   cd quant_trader
   ```

2. **.env 파일 만들기**  
   ```bash
   nano .env
   ```  
   - 아래 내용을 **그대로** 붙여넣되, `여기에_업비트_Access_Key` / `여기에_업비트_Secret_Key` 부분만 **본인 키로** 바꾸세요.  
   ```env
   UPBIT_ACCESS_KEY=여기에_업비트_Access_Key
   UPBIT_SECRET_KEY=여기에_업비트_Secret_Key
   ```  
   - **Ctrl+O** 누른 뒤 Enter (저장)  
   - **Ctrl+X** (나가기)

---

## 4단계: 한 번에 설정하고 대시보드 실행하기

아래 **한 줄**만 복사해서 붙여넣고 Enter 하세요.

```bash
bash scripts/setup-server.sh
```

- Node 설치, 패키지 설치, 빌드, PM2로 실행까지 자동으로 됩니다.  
- 끝나면 `✅ 설정 끝` 이라고 나옵니다.  
- 그 다음에 **`sudo env PATH=...` 로 시작하는 한 줄**이 나오면, 그 줄을 **통째로 복사**해서 한 번 더 입력하고 Enter 하세요. (서버 재부팅 후에도 대시보드가 자동으로 켜지게 하는 설정입니다.)

---

## 5단계: AWS에서 3003 포트 열기

지금은 서버 안에서만 대시보드가 돌고 있어서, **밖(본인 PC)에서 접속하려면** AWS에서 포트를 열어야 합니다.

1. **AWS 콘솔** 로그인 → **EC2** → 왼쪽 메뉴 **보안 그룹(Security Groups)**

2. 인스턴스에 붙어 있는 보안 그룹 선택 (이름에 `launch-wizard` 또는 `Quant` 등이 있을 수 있음)  
   - **인바운드 규칙** 탭 → **인바운드 규칙 편집**

3. **규칙 추가**  
   - **유형**: 사용자 지정 TCP  
   - **포트 범위**: 3003  
   - **소스**: 내 IP (또는 0.0.0.0/0 — 어디서나 접속 가능, 보안은 약해짐)  
   - **규칙 저장**

4. 브라우저에서 아래 주소로 접속해 보기:  
   **http://15.135.65.106:3003**  
   - 업비트 연동·잔고가 보이면 성공입니다.

## 자주 나오는 상황

- **"Permission denied"**  
  - .pem 파일이 있는 폴더에서 `ssh -i "my-quant-key.pem" ubuntu@15.135.65.106` 실행했는지 확인  
  - 키 파일 이름이 정확한지 확인 (my-quant-key.pem)

- **접속은 되는데 3003 접속이 안 됨**  
  - AWS 보안 그룹에서 **인바운드 3003** 포트를 열었는지 확인

- **서버 재부팅 후에도 대시보드가 돌아가나요?**  
  - 4단계에서 PM2로 실행했기 때문에, 재부팅 후에도 자동으로 다시 켜지도록 되어 있습니다. (한 번 `pm2 startup` 안내가 나왔을 때 나온 명령을 실행해 두면 더 안전합니다.)

진행하다가 **어느 단계에서 막히는지** (예: 2단계 SSH, 4단계 스크립트, 5단계 포트)만 알려주시면, 그 부분만 더 쉽게 풀어서 적어 드리겠습니다.
