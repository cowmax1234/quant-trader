# VPS 고정 IP 설정 가이드 — AWS (EC2 + Elastic IP)

본인 명의 **AWS**로 EC2 서버 하나 만들고, **Elastic IP(고정 IP)**를 붙여서 업비트 Open API에 등록하는 순서입니다.

> **참고:** AWS 프리티어는 가입 후 12개월 동안 t2.micro 등 무료 한도가 적용됩니다. 12개월 이후에도 인스턴스를 쓰면 소량 과금될 수 있습니다. Elastic IP는 **인스턴스에 연결해 두면** 무료이고, 연결 안 하고 두면 요금이 부과됩니다.

---

## 1. AWS 가입 (개인 계정)

1. **https://aws.amazon.com/ko/** 접속 → **AWS 계정 생성**
2. 이메일, 비밀번호, 결제 수단(카드) 등록 (프리티어 한도 내 사용 시 과금 없음)
3. 가입 후 **AWS Management Console** 로그인

---

## 2. EC2 인스턴스(서버) 만들기

1. 상단 검색창에 **EC2** 입력 → **EC2** 서비스로 이동
2. 왼쪽 메뉴 **인스턴스** → **인스턴스 시작** 클릭

3. **이름**: `quant-trader` (원하는 이름)

4. **애플리케이션 및 OS 이미지**  
   - **Ubuntu Server 22.04 LTS** 선택  
   - (프리티어 사용 시 해당 이미지가 **무료 사용 가능**으로 표시되는지 확인)

5. **인스턴스 유형**  
   - **t2.micro** 선택 (프리티어 12개월 무료)

6. **키 페어(로그인)**  
   - **새 키 페어 생성**  
   - 키 페어 이름: `quant-trader-key`  
   - **.pem 키 페어 다운로드** → 파일을 안전한 곳에 보관 (다시 받을 수 없음)

7. **네트워크 설정**  
   - **편집** 클릭  
   - **퍼블릭 IP 자동 할당**: 활성화(기본값)  
   - **보안 그룹**: 새로 만들기 또는 기존 것 사용  
   - 규칙 추가: **SSH(22)** — 내 IP 또는 0.0.0.0/0 (나중에 VPS에서만 접속하려면 내 IP만 허용 권장)

8. **스토리지**: 기본 8GB 또는 필요 시 30GB 이내 (프리티어 범위)

9. **인스턴스 시작** 클릭

10. 인스턴스 목록에서 상태가 **실행 중**이 될 때까지 대기 (1~2분)

---

## 3. Elastic IP(고정 IP) 할당하고 EC2에 연결

EC2만 만들면 **재시작 시 공인 IP가 바뀔 수** 있습니다. **Elastic IP**를 쓰면 고정 IP를 업비트에 등록할 수 있습니다.

1. EC2 왼쪽 메뉴 **네트워크 및 보안** → **탄력적 IP**

2. **탄력적 IP 주소 할당** 클릭  
   - 설정은 기본값 그대로 → **할당** 클릭

3. 목록에 생긴 **탄력적 IP 주소**를 복사해 둠 (예: `3.35.xxx.xxx`)  
   → **이 주소를 업비트 Open API에 등록합니다.**

4. **작업** → **탄력적 IP 주소 연결**

5. **리소스 유형**: 인스턴스  
   **인스턴스**: 방금 만든 `quant-trader` 선택  
   **프라이빗 IP 주소**: (자동 선택된 것 그대로)  
   → **연결** 클릭

6. (선택) 인스턴스에 원래 붙어 있던 **퍼블릭 IP**는 이제 쓰이지 않으므로, 인스턴스 상세에서 **퍼블릭 IPv4 주소**가 방금 연결한 **Elastic IP**로 보이면 정상입니다.

> **주의:** Elastic IP를 **인스턴스에 연결하지 않고** 두면 요금이 부과됩니다. 반드시 위처럼 인스턴스에 연결해 두세요.

---

## 4. 업비트 Open API에 고정 IP 등록

1. **PC**에서 [업비트 → 마이페이지 → Open API 관리](https://upbit.com/mypage/open_api_management) 접속

2. API 키 **생성** 또는 **편집**

3. **IP 제한**에 위에서 복사한 **Elastic IP 주소** 한 개만 입력 (예: `3.35.xxx.xxx`)

4. 저장

이제 업비트 API는 **이 Elastic IP에서 나간 요청만** 허용합니다.

---

## 5. (참고) 회사 PC에서 대시보드 쓰는 경우

업비트에 **VPS(Elastic IP)만** 등록해 두면, **회사 PC에서** 대시보드를 돌릴 때 업비트 API 호출이 **거절**됩니다 (IP 불일치).

선택지:

- **개발용 API 키**: IP 제한 **없음** → 회사 PC에서 연동·테스트  
- **실거래용 API 키**: **Elastic IP만** 등록 → VPS에서 봇/대시보드 24시간 실행할 때 사용  

지금은 개발용 키(IP 제한 없음)로 대시보드 연동하고, VPS 준비되면 실거래용 키(Elastic IP만)를 VPS 서버의 `.env`에 넣어 쓰는 흐름을 추천합니다.

---

## 6. (나중에) 이 서버에 SSH 접속·봇 배포

- **SSH 접속 예시** (Windows: PowerShell 또는 WSL, 키 경로와 IP는 본인 값으로 변경):

```bash
# 키 권한 설정 (WSL/Git Bash 등)
chmod 400 quant-trader-key.pem

# 접속 (Ubuntu 기본 사용자명: ubuntu)
ssh -i "quant-trader-key.pem" ubuntu@3.35.xxx.xxx
```

- 이 서버에 Node.js 설치 후 `quant-trader` 프로젝트 복사/클론 → `.env` 설정 → `npm run build` 후 `npm start` 또는 PM2로 24시간 실행하면 됩니다.
- 상세 배포(방화벽, 포트, PM2 등)는 필요할 때 별도로 정리할 수 있습니다.

---

## 요약

| 단계 | 할 일 |
|------|--------|
| 1 | AWS 개인 가입 |
| 2 | EC2 인스턴스 생성 (Ubuntu 22.04, t2.micro), 키 페어(.pem) 저장 |
| 3 | 탄력적 IP 할당 → EC2 인스턴스에 연결 (고정 IP 확보) |
| 4 | 그 고정 IP를 업비트 Open API 관리에 등록 |
| 5 | (선택) 개발용 키는 IP 제한 없이, 실거래용 키만 Elastic IP로 제한 |
| 6 | (나중에) SSH로 접속해서 봇/대시보드 배포 |

이렇게 하면 **고정 IP 한 개**만 업비트에 등록해 두고, 실제 주문은 이 AWS 서버에서만 나가게 할 수 있습니다.
