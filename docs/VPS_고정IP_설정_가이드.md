# VPS 고정 IP 설정 가이드 (업비트 API용)

본인 명의 **Oracle Cloud Free Tier**로 서버 하나 만들고, **고정 IP**를 받아서 업비트 Open API에 등록하는 순서입니다.  
(24시간 켜 둘 컴퓨터가 없을 때, 나중에 봇도 이 서버에서 돌릴 수 있습니다.)

---

## 1. Oracle Cloud 가입 (개인 계정)

1. **https://www.oracle.com/kr/cloud/free/** 접속
2. **무료 체험 시작** → 본인 명의로 가입 (이메일, 비밀번호, 결제 수단 등록 필요하지만 **Free Tier는 사용량 한도 내 무료**)
3. 가입 후 **Cloud Console** 로그인

---

## 2. VM 인스턴스(서버) 만들기

1. 콘솔 메뉴: **컴퓨팅(Compute)** → **인스턴스(Instances)**
2. **인스턴스 만들기(Create instance)** 클릭
3. 설정 예시:
   - **이름**: `quant-trader` (아무거나 가능)
   - **이미지**: **Change image** → **Canonical Ubuntu 22.04** 선택 (**Always Free** 마크 있는 것)
   - **Shape**: **Change shape** → **AMD** → **VM.Standard.E2.1.Micro** (Always Free)
   - **SSH 키**: **Generate a key pair for me** 선택 후 **개인 키 저장** 버튼으로 `.key` 파일 다운로드 (절대 잃어버리지 말 것)
4. **만들기(Create)** 클릭
5. 인스턴스 상태가 **Running**이 될 때까지 대기 (몇 분)
6. 인스턴스 상세 화면에서 **공용 IP 주소(Public IP)** 한 개가 보이면 일단 메모 (나중에 고정 IP로 바꿀 예정)

---

## 3. 고정 IP(예약 공용 IP) 만들고 서버에 붙이기

지금 붙어 있는 건 **임시(Ephemeral)** IP라, 인스턴스를 지우거나 재생성하면 바뀔 수 있습니다. **예약(Reserved)** IP로 바꿔야 업비트에 안심하고 등록할 수 있습니다.

1. **네트워킹(Networking)** → **IP 관리(IP Management)** → **예약된 공용 IP(Reserved Public IPs)**
2. **예약된 공용 IP 주소 생성(Create reserved public IP)** 클릭
   - 이름: `quant-trader-ip`
   - 구획(Compartment): 본인 테넌시 기본 구획
   - **생성** 클릭
3. 생성된 **예약된 공용 IP 주소**를 복사해 둠 (예: `132.145.xxx.xxx`) → **이걸 업비트에 등록할 주소**

이제 이 IP를 VM에 붙입니다.

4. **컴퓨팅** → **인스턴스** → 방금 만든 인스턴스 클릭
5. **리소스(Resources)** 왼쪽에서 **연결된 VNIC(Attached VNIC)** → **기본 VNIC** 클릭
6. **IPv4 주소(IPv4 Addresses)** 쪽에서 **편집(Edit)** 또는 해당 공용 IP 행의 메뉴(⋮) 사용
7. 현재 **임시 공용 IP**가 있으면:
   - **공용 IP 없음(None)** 으로 변경 후 저장
   - 다시 **편집** → **예약된 공용 IP(Reserved public IP)** 선택
   - 방금 만든 `quant-trader-ip` 선택 → 저장

이제 이 VM의 공용 IP는 **예약된 고정 IP** 하나뿐입니다. 이 주소를 업비트에 등록하면 됩니다.

---

## 4. 업비트 Open API에 고정 IP 등록

1. **PC**에서 [업비트 → 마이페이지 → Open API 관리](https://upbit.com/mypage/open_api_management) 접속
2. API 키 생성 또는 기존 키 **편집**
3. **IP 제한** 항목에 위에서 적어 둔 **예약된 공용 IP** 한 개만 입력 (예: `132.145.xxx.xxx`)
4. 저장

이제 **실제 API 호출(주문·잔고 조회 등)은 이 고정 IP를 쓰는 서버에서만** 업비트가 허용합니다.

**주의:** 지금처럼 회사 PC에서 대시보드(`localhost:3003`)를 돌리면서 업비트 API를 호출하면, 업비트에는 **회사 PC의 공인 IP**로 찍힙니다. 그래서 **업비트에 VPS 고정 IP만 등록해 두면, 회사 PC에서는 업비트 API가 거절**됩니다. 선택지는 세 가지입니다.

- **A)** 업비트에서 **API 키를 두 개** 만든다: (1) 개발용 — IP 제한 없음 → 회사 PC에서 연동 테스트, (2) 실거래용 — VPS 고정 IP만 등록 → 나중에 VPS에서 봇/대시보드 돌릴 때 사용.
- **B)** VPS 고정 IP만 등록하고, **대시보드/봇은 처음부터 VPS에만** 설치해서, 그 서버에서만 업비트 API 호출 (회사 PC에서는 코드만 편집).
- **C)** 임시로 업비트에 **회사 PC 공인 IP + VPS 고정 IP** 둘 다 등록(가능한 경우)하고 개발하다가, 나중에 VPS만 쓰면 회사 IP는 삭제.

원하면 A처럼 개발용 키(IP 제한 없음)로 지금 대시보드 연동만 먼저 하고, VPS 준비된 뒤 실거래용 키(VPS IP만)를 쓰는 흐름을 추천합니다.

---

## 5. (참고) 이 서버에 나중에 봇/대시보드 배포하기

- 이 VM에 SSH로 접속한 뒤, Node.js 설치하고 이 `quant-trader` 프로젝트를 클론/복사해서 `npm run build` → `npm start` (또는 PM2 등으로 24시간 실행) 하면 됩니다.
- 환경 변수(`.env`)는 이 서버에만 두고, API 키는 여기서만 사용하면 됩니다.
- 상세 배포 단계(SSH 접속, Node 설치, 포트 열기 등)는 필요할 때 별도 가이드로 정리할 수 있습니다.
